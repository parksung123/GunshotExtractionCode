1. 사용자가 직접 추출할 수 있는 기능을 check 에서 fix 로 봐꿈 
2. fix 기능이 입력값일 경우 그 다음 입력값을 봐꿈 
- 전 코드는 총성 한개씩만 추출하게되있음 -> 현재 코드는 여러 총성을 한번에 추출가능 
- 원래 직접 봐꿀 총성 갯수와 추출할 n번째 총성을 입력해야됬지만 이젠 추출할 n번째 총성 리스트를 입력하면 끝 
ex) 3번째와 2번째 총성 추출을 한다면 :
예전 코드 : read_wave(path, plot, 'fix', 1, 2) 한 후 read_wave(path, plot, 'fix', 1, 3) 
현재 코드: read_wave(path, plot, 'fix', [2 3])
3. 전 코드는 fix를 위해 시작점을 플롯에 배정할시 새로운 플롯이 나왔서 시간 소모. 이젠 시작점+끝점 쌍으로 같은 플롯에서 배정 가능 
4. 입력값 테스트 코드 추가
- algorithm : 'cc', 'xf', 또는 'fix' 가 아니면 에러메세지 출력
- plot : 0 또는 1 이 아니면 에러메세지 출력
- nthRounds : 리스트 또는 숫자가 아니면 에러메세지 출력 / 리스트에 음수가 있으면 에러메세지 출력
- channel: 숫자가 아니거나 음수이면 에러메세지 출력
- directory: 존재하질 않을 경우 에러메세지 출력
5. fix 기능 이용시 시작점이 끝점보다 크면 다시 시도 가능
6. 사용자가 양수가 아닌 총성 갯수 값을 입력하면 다시 시도 가능
